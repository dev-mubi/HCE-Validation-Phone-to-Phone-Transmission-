# HCE Transaction Reporter - Backend Service

This directory contains the lightweight FastAPI backend server that handles transaction reports generated by the HCE Reader client and dispatches email confirmations using the **Resend API**.

---

## 🚀 Features

- **Pydantic Validation**: Strict schema verification for incoming payload, latitude/longitude, and ISO 8601 timestamps.
- **Permissive CORS**: Configured to accept incoming connections from physical mobile clients on the local area network.
- **Resend SDK Integration**: Dispatches transaction alerts to designated validation teams.
- **Google Maps lookup**: Automatically generates a clickable coordinate tracking link (`https://www.google.com/maps?q=<lat>,<lng>`) within the email.
- **Clean Monospaced Design**: Email layout uses an editorial technical structure in line with the project's visual layout principles.

## 🛠️ Prerequisites & Setup

### Requirements
- **Python**: Version `3.10` or higher.
- **Resend Account**: Active API key and a verified sending domain (or use the default sandbox configuration).

### Step-by-Step Installation
1. Change directory to `/server`:
   ```bash
   cd server
   ```
2. Initialize and activate a virtual environment:
   ```bash
   python -m venv venv
   # On Windows (PowerShell):
   .\venv\Scripts\activate
   # On macOS/Linux:
   source venv/bin/activate
   ```
3. Install package dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Configure environment variables:
   Copy `.env.example` to `.env`:
   ```bash
   cp .env.example .env
   ```
   Open `.env` and fill in your keys:
   - `RESEND_API_KEY`: Your active Resend token (`re_...`).
   - `RESEND_SENDER_EMAIL`: Set to `onboarding@resend.dev` (sandbox mode) or your custom verified domain sender address (e.g. `alerts@yourdomain.com`).
   - `REPORT_RECIPIENT_EMAIL`: Target email to receive transaction reports.

## 💻 Running the Server

Start the development server with Uvicorn:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
- Hosting on `0.0.0.0` ensures the server is reachable by physical devices connected to the same local network (LAN).
- The service will run at: `http://localhost:8000`.
- Interactive OpenAPI documentation can be viewed at: `http://localhost:8000/docs`.

## 📡 API Reference

### Service Health Check
`GET /health`

Used to check server uptime and keep Render web services awake via automated ping services or cron jobs.

#### Response
```json
{
  "status": "healthy",
  "timestamp": "ISO 8601 current timestamp"
}
```

### Post Transaction Log
`POST /transaction-report`

#### Request Headers
```http
Content-Type: application/json
```

#### Request JSON Body Schema
```json
{
  "payload": "string",
  "latitude": 0.0,
  "longitude": 0.0,
  "timestamp": "ISO 8601 timestamp string (e.g. '2026-06-21T09:00:00Z')"
}
```

#### Curl Test Command
To verify your server configuration locally:
```bash
curl -X POST "http://localhost:8000/transaction-report" \
     -H "Content-Type: application/json" \
     -d '{"payload":"MOCK_PAYLOAD_TEST","latitude":37.7749,"longitude":-122.4194,"timestamp":"2026-06-21T09:00:00Z"}'
```

#### Response (Success)
```json
{
  "status": "SUCCESS",
  "message": "Transaction reported and email dispatched."
}
```
